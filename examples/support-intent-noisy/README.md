# Support Intent (Noisy)

Classify messy customer support emails into intent and sentiment categories. Emails contain OCR artifacts, mobile typos, mixed languages, and corrupted encodings — the kind of noise real support inboxes deal with.

## Categories

**Intent** (7 classes): refund_request, setup_help, billing_question, technical_issue, feedback, account_change, shipping_inquiry

**Sentiment** (4 classes): angry, frustrated, neutral, satisfied

## Data

| Set | Files | Purpose |
|-----|-------|---------|
| `data/emails/` | 55 raw emails | Source material for synthetic training data |
| `data/eval-emails/` | 14 raw emails | Source material for evaluation set |

The raw emails are uploaded to S3. Slemify uses Bedrock to generate 800 synthetic training pairs and 100 evaluation pairs from these source files.

## Results

The [latest report](report.html) shows:

- **95% intent accuracy** across 100 evaluation samples
- **p50 latency: 1510ms** on CPU (Graviton Spot)
- **$117/mo** per replica (Spot), serving ~0.7 req/sec
- **100% consistency** on repeated identical inputs

The fine-tuned SLM produces structured pipe-delimited output (`intent|sentiment`) that downstream systems parse directly, while a zero-shot LLM outputs verbose markdown that requires additional extraction.

## Usage

```bash
# 1. Upload data to S3
./upload-to-s3.sh

# 2. Deploy the full pipeline
slemify deploy --config expert.yaml

# 3. View the report
slemify report --config expert.yaml
```

## Config

See [expert.yaml](expert.yaml) for the full configuration.

Key choices:
- **Base model**: a 3B parameter model — good multilingual support for mixed-language emails
- **Quantization**: q4_k_m — 4-bit quantization for CPU inference
- **Synthetic pairs**: 800 training + 100 eval, generated by Claude Sonnet
- **Training**: Spot GPU with automatic checkpoint recovery
